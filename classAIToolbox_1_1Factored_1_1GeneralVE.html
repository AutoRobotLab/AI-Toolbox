Detailed Description

template<typename Implementation, typename E>
class AIToolbox::Factored::GeneralVE< Implementation, E >

This class represents the Variable Elimination process.

This class performs variable elimination on a factor graph. It first builds the graph starting from a list of rules. These rules are grouped by the agents they affect, and each group is added to a single factor connected to those agents.

Each agent is then eliminated from the graph, and all rules connected to it are processed in order to find out which action the agent being eliminated should take.

When all agents have been eliminated, only the optimal rules containing the best actions remain. The ones that provide the best reward are joined into a single Action, which is then returned.

The complexity of this method in the induced width of the graph; i.e. it is exponential in the maximum number of agents found attached to the same factor at any point of the elimination process. This number could be higher than in the original input, as the elimination process can create bigger factors than the initial ones.

In any case, given that each factor is usually linked to few agents, and that this process allows avoiding considering the full factored Action at any one time, it is usually much faster than a brute-force approach.

This class performs the process but relies on classes to derive it using C++ CRTP mechanism. Additionally, this class must be passed the Entry type as it stores it, and there's no way to avoid it.

A set of functions must be defined by the child in order to work. While they are described below, there's no alternative at looking at the code to see exactly how and when they are called. 

void setup();
void removalSetup(size_t);

These functions are hooks called at the beginning of the Variable Elimination process (just after input insertion), and at the beginning of each agent removal respectively. They don't have to do anything, but may be useful in order to do things or compute values needed by the deriving class. 

Entry makeEntry();
Entry makeEntry(const InputRule &);

These functions must create a valid Entry, either from scratch or from one of the input rules (which here we don't know what they are).

We only assume that the input rule contains a "a_" field which we use to index the underlying graph. 

Reductor makeReductor(const PartialAction &, size_t agentId);
void updateReductor(Reductor &, const Rules &, const PartialAction &);
void outputReductor(Entry &, Reductor &&);

These function create a Reductor instance to process rules during agent elimination. The type of the Reductor is up to the user, and it is not known by this class. The reductor must finally be used to update the Entry for the current local action. 

bool isEntryValid(Entry &);

This function may edit the entry, but must return whether it can be accepted for inclusion in the Rule for the current local action (independent of the agent being eliminated). 

void updateFactorRules(Rules &, Rules &&);

This function must insert the produced rules into the rules for the factor of the remaining agent. 

FinalReductor makeFinalReductor();
void updateFinalReductor(FinalReductor &, Entries &&);
Result outputFinalReductor(FinalReductor &&);

This functions create a FinalReductor to process all the Entries that have been left from the Variable Elimination procedure. The FinalReductor can then be further processed to transform it into the returned result.
